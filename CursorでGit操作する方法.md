# 🖱️ CursorでGit操作する方法（初心者向け）

CursorにはGit機能が統合されているので、ターミナルを使わずにGUIでGit操作ができます！

## 📋 基本的な操作手順

### ステップ1: ソース管理パネルを開く

1. Cursorの左側にある「**ソース管理**」アイコン（分岐マーク）をクリック
   - または、`Cmd + Shift + G`（Mac）を押す

### ステップ2: 変更されたファイルを確認

ソース管理パネルに変更されたファイルが表示されます：
- **変更済み**（M）: 既存ファイルが変更された
- **新規**（U）: 新しいファイルが追加された
- **削除**（D）: ファイルが削除された

### ステップ3: ファイルをステージング（追加）

#### 方法1: 個別に追加

1. ファイル名の横にある「**+**」ボタンをクリック
2. または、ファイル名を右クリック → 「**ステージング**」を選択

#### 方法2: すべてを追加

1. 「**変更**」セクションの上にある「**+**」ボタンをクリック
2. または、「**すべての変更をステージング**」をクリック

### ステップ4: コミット（変更を記録）

1. 上部の「**メッセージ**」欄にコミットメッセージを入力
   - 例: "Vercel設定を追加"
   - 例: "READMEを更新"
   - 例: "バグを修正"

2. 「**✓ コミット**」ボタンをクリック
   - または、`Cmd + Enter`（Mac）を押す

### ステップ5: GitHubにアップロード（プッシュ）

1. ソース管理パネルの下部にある「**...**」（3つの点）をクリック
2. 「**プッシュ**」を選択
   - または、「**同期変更**」をクリック（プッシュとプルを同時に実行）

## 🎯 実践例

### 例1: 1つのファイルを更新した場合

1. ソース管理パネルを開く（`Cmd + Shift + G`）
2. 変更されたファイルの横の「**+**」をクリック
3. メッセージ欄に「READMEを更新」と入力
4. 「**✓ コミット**」をクリック
5. 「**...**」→「**プッシュ**」をクリック

### 例2: 複数のファイルを更新した場合

1. ソース管理パネルを開く
2. 「**すべての変更をステージング**」をクリック
3. メッセージ欄に「Vercel設定と手順書を追加」と入力
4. 「**✓ コミット**」をクリック
5. 「**...**」→「**プッシュ**」をクリック

## 🔍 便利な機能

### 変更内容を確認

1. ファイル名をクリック
2. 変更内容が表示されます（追加された行は緑、削除された行は赤）

### コミット履歴を確認

1. ソース管理パネルの「**...**」をクリック
2. 「**コミット履歴を表示**」を選択

### ブランチを切り替え

1. 下部のステータスバーでブランチ名をクリック
2. ブランチを選択

## ⚠️ 注意事項

### 初回プッシュ時

初回プッシュ時は、GitHubの認証情報を入力する必要がある場合があります。

### エラーが出た場合

- **認証エラー**: GitHubの認証情報を確認
- **競合エラー**: 他の場所で変更があった場合は、まず「**プル**」を実行

## 📝 よく使うショートカット

- `Cmd + Shift + G`: ソース管理パネルを開く
- `Cmd + Enter`: コミット
- `Cmd + K, Cmd + Enter`: コミットしてプッシュ

## 🎉 完了！

これでCursor内でGit操作ができます。ターミナルを使わなくてもOKです！
